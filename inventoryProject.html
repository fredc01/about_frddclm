Inventory Management & Request Allocation System

A complete system for tracking inventory items, managing allocation requests, and monitoring warranties.

Key Features

📦

Inventory Lifecycle Management

Track items from purchase to warranty expiration with detailed history logs.

🔄

Dynamic Allocation System

Users can request items, which go through a structured approval and deployment workflow.

👥

Role-Based Views

Admins, Approvers, and Requestors see tailored interfaces suited to their responsibilities.

🖼️

Image & Document History

Upload and view photos and documents associated with each inventory item.

⏳

Automated Warranty Tracking

Visual indicators and updates for warranty expiry and renewal tracking.

🖥️

Modern UI/UX

Crafted with Bootstrap and custom styles for a clean, responsive, and user-friendly interface.

Inventory Items - API Examples

GET - Fetch All Inventory Items

GET /api/inventory

Response:
[
  {
    "id": 1,
    "itemName": "Dell Laptop",
    "serialNumber": "SN12345678",
    "purchaseDate": "2023-01-15",
    "warrantyExpiry": "2025-01-15",
    "status": "Available"
  }
]

POST - Create New Inventory Item

POST /api/inventory

Request Body:
{
  "itemName": "HP Monitor",
  "serialNumber": "SN98765432",
  "purchaseDate": "2024-02-20",
  "warrantyExpiry": "2026-02-20",
  "status": "Available"
}

PUT - Update Inventory Item

PUT /api/inventory/{id}

Request Body:
{
  "itemName": "HP Monitor (Updated)",
  "status": "Allocated"
}

DELETE - Remove Inventory Item

DELETE /api/inventory/{id}

Allocation Request Process

A system to handle item allocation, approvals, and history tracking for users and admins.

Allocation Requests - API Examples

GET - Fetch All Requests

GET /api/requests

Response:
[
  {
    "id": 101,
    "itemId": 1,
    "requestedBy": "John Doe",
    "status": "Pending",
    "requestDate": "2025-04-15",
    "approvalDate": null
  }
]

POST - Submit New Request

POST /api/requests

Request Body:
{
  "itemId": 1,
  "requestedBy": "John Doe"
}

PUT - Approve or Reject Request

PUT /api/requests/{id}

Request Body:
{
  "status": "Approved",
  "approvalDate": "2025-04-20"
}

DELETE - Cancel Request

DELETE /api/requests/{id}
